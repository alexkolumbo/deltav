# Delta V — децентрализованная AI-сеть

[English](README.md) · **Русский**

Лёгкий PoS-блокчейн + сеть GPU-нод + умный роутинг open-source моделей с
[HuggingFace](https://huggingface.co/models). Пользователь шлёт запрос в
OpenAI/Ollama/Anthropic-совместимый шлюз; сеть сама выбирает лучшую
модель, которая физически влезает в VRAM живой ноды, исполняет инференс,
записывает чек в чейн, платит ноде токенами DVT и выборочно перепроверяет
её честность.

```
клиент ──► шлюз (/v1/chat/completions · /api/chat · /v1/messages)
              │  SmartRouter: модель ⨯ нода (VRAM-fit, репутация, стейк, цена, загрузка)
              ▼
        демон ноды ──► ComputeBackend (llama.cpp/Vulkan на AMD·NVIDIA·Intel·CPU; Groq; …)
              │
              ▼
        Delta V chain: INFERENCE_RECEIPT (оплата) ──► SPOT_CHECK (перепроверка / слэшинг)
```

Vendor-agnostic: один GGUF работает на **AMD, NVIDIA, Intel, Apple или
CPU** через llama.cpp (Vulkan не требует CUDA/ROCm). Отдаются только
**непроприетарные** модели — Qwen, Llama, Gemma, Mistral, DeepSeek,
открытые веса Grok от xAI или ваша собственная. Форма API — это слой
трансляции, не про модель; сеть говорит на формах, которые использует
open-source экосистема: **OpenAI** (стандарт де-факто — Hermes,
grok-build, goose, opencode), **Ollama** (локальные инструменты) и
**Anthropic** (доп. форма для клиентов на `anthropic`-SDK).

## Нода за 5 минут (для всех) — `deltav setup`

Двуязычный (RU/EN) мастер проведёт от чистой машины до живой
зарабатывающей ноды, объясняя каждый шаг. Определит железо, **скачает
готовый бинарник llama.cpp** под вашу ОС+GPU (Vulkan для AMD/NVIDIA/Intel,
Metal для Apple, CPU-фолбэк — без компиляции), подберёт и скачает модель,
заведёт кошелёк, подключит к сети, посчитает цену и всё запустит.

```bash
# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/alexkolumbo/deltav/main/install.sh | sh
# Windows (PowerShell)
irm https://raw.githubusercontent.com/alexkolumbo/deltav/main/install.ps1 | iex

# или, если Python 3.11+ уже есть:
deltav setup --seed http://<любая-живая-нода>:9100
```

Можно принять рекомендованную модель или вставить свой HuggingFace-репо —
с **анализом** влезаемости (вердикт + макс. контекст) или **форсированно**.

**Внешний доступ — автоматически.** `deltav join` сам определяет, доступен
ли узел напрямую, а за NAT/CGNAT открывает исходящий туннель через любой
публичный узел-релей — доступен всей сети **без проброса портов, без ручного
IP, без настройки TLS** и без стороннего сервиса (релеи — это обычные узлы).
См. [Хостинг → Сделать доступной](docs/HOSTING.ru.md).

## Подключение клиента

Шлюз — полноценный **OpenAI-совместимый** эндпоинт со стримингом (SSE),
tool calling и биллингом, плюс формы **Ollama** и **Anthropic**.

| Клиент ждёт | Указать на | Кто |
|---|---|---|
| OpenAI | `http://<gw>:9000/v1` | Hermes, grok-build, goose, opencode, openai SDK |
| Ollama | `http://<gw>:9000` | Open WebUI, LangChain-Ollama, десктоп-клиенты |
| Anthropic | `http://<gw>:9000` | любой клиент на `anthropic`-SDK |

**Полные руководства**: [Хостинг ноды](docs/HOSTING.ru.md) ([EN](docs/HOSTING.md)) ·
[Использование как клиент](docs/CLIENT.ru.md) ([EN](docs/CLIENT.md)) ·
рецепты по инструментам и SDK — в [docs/INTEGRATIONS.md](docs/INTEGRATIONS.md).

**Companion** (`deltav companion`, `/v1/companion/*`) — персональный агент
с собственным слоем памяти и самоулучшением при **строгой изоляции
юзеров**: личность берётся из вашего ключа, поэтому один пользователь
никогда не доберётся до памяти другого. Каждый ход вспоминает ваши факты
и выученное, а после — рефлексирует и запоминает новое, так что даже
слабые модели со временем работают лучше именно для вас.

## Типы моделей

Помимо текста сеть поддерживает (groundwork, работает через mock,
реальный движок реализует один интерфейс `ComputeBackend`):

- **мультимодальные (vision)** — модель принимает изображения на вход
  (`ModelSpec.vision`, `InferRequest.images`); в каталоге Qwen2.5-VL;
- **диффузия (текст→изображение)** — `kind="image"`, OpenAI-совместимый
  `POST /v1/images/generations`, роутинг на диффузные ноды, чек и
  спот-чек как у текста; в каталоге Stable Diffusion 1.5 и FLUX.1
  (через stable-diffusion.cpp / GGUF);
- **открытые веса Grok** от xAI (большие — для мощных нод);
- **собственная модель сети + RL-обучение** — задел в `deltav/training/`:
  каждый чек = пара (промпт, вывод), каждый спот-чек = сигнал
  награды (verified/slashed) → датасет и reward уже есть; интерфейс
  `TrainingCoordinator` определяет раунд обучения для мощной ноды.

## Токеномика

Цена заякорена в себестоимости: **держатель ноды окупает электричество и
получает +50% за сервис**. Коэффициент электричества — среднемировая цена
($0.155/кВт·ч); каждый оператор считает свою (`deltav price`) и ставит на
рынок (`--price`). Референс-пег: 150 Вт / 30 tok/s → $0.32/1M → дефолт
10 udvt/токен → **1 DVT ≈ $0.032**.

**Chain pool**: 10% каждого платежа копится в пуле, каждую эпоху
распределяется — 30% в фонд разработчиков, остальное нодам пропорционально
отработанным токенам. **Биллинг**: API-ключ `dvk_…` — это он-чейн
кошелёк; пополняете адрес, каждый запрос оплачивается с него.

## Доверие и консенсус

- **Авторизация оплаты**: нода списывает только с подписью реквестера над
  `(request_hash, node, model, price_limit)`.
- **Spot-check**: валидаторы перепрокатывают долю джобов и сверяют хэш;
  ложь → сжигание стейка и репутации (для недетерминированных бэкендов —
  fuzzy-сверка).
- **Liveness**: RANDAO-рандомизация пропозера, запасные слоты, джейл за
  прогулы.
- **Лёгкий клиент** (`deltav verify`): по доверенному генезису проверяет
  целостность чейна, кворум нод и что каждое списание с вашего ключа было
  им авторизовано.
