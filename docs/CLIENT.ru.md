# Использование Delta V (клиент)

[English](CLIENT.md) · [Русский](CLIENT.ru.md)

Вы обращаетесь к сети через **шлюз** — эндпоинт, совместимый с
OpenAI/Ollama/Anthropic. Выберите путь под задачу; самый простой — первый.

Понадобится: адрес шлюза (напр. `http://10.0.0.223:9000`) и, для оплаты,
API-ключ. Модель всегда `auto` (сеть выберет лучшую).

## 1. Веб-приложение — ничего не ставить

Откройте **`http://<шлюз>:9000/chat`** в любом браузере (телефон или ноут):

- Чат со стримингом, плюс режимы **Агент** (веб-поиск + инструменты),
  **Сворм** (несколько моделей параллельно) и **Компаньон** (помнит вас).
- **🖼 загрузка изображений** для vision-моделей.
- **⚙ настройки** — один раз задать адрес шлюза и ваш ключ.
- На iOS/Android: *Поделиться → На экран «Домой»* — установится как
  приложение.

## 2. Готовые инструменты (goose, Open WebUI, opencode, ваш код)

Укажите инструменту адрес шлюза:

| Инструмент говорит | Base URL | Ключ |
|---|---|---|
| OpenAI (goose, opencode, openai SDK, Hermes, grok-build) | `http://<gw>:9000/v1` | ваш `dvk_…` |
| Ollama (Open WebUI, LangChain, десктоп) | `http://<gw>:9000` | ваш `dvk_…` |
| Anthropic (`anthropic` SDK) | `http://<gw>:9000` | ваш `dvk_…` |

Модель: `auto`. Подробные рецепты: [INTEGRATIONS.md](INTEGRATIONS.md).

## 3. Zero-config — прокси `deltav serve`

Чтобы инструменту **не нужен был ни ключ, ни настройка** — указывайте на
`localhost`:

```bash
pip install "deltav-network @ git+https://github.com/alexkolumbo/deltav"
deltav connect --url http://<gw>:9000 --key dvk_…   # сохранить один раз
deltav serve                                          # прокси на :11434
```

Теперь любой инструмент работает без ключа:
- Ollama: `http://localhost:11434`
- OpenAI: `http://localhost:11434/v1`
- Anthropic: `http://localhost:11434`

Прокси подставит ключ и сделает failover по шлюзам.

## 4. Командная строка

```bash
deltav repl                       # интерактивный чат со стримингом; /agent /swarm /model
deltav chat "Что такое delta-v"   # разово
deltav agent "найди последнюю версию llama.cpp и посчитай 2**20"
deltav swarm "сравни два подхода" --mode vote -n 3
deltav companion                  # персональный агент, который вас помнит
deltav models / network / balance dv1… / send --to dv1… --amount 100
```

## 5. Telegram — из любой точки

В сети можно поднять **Telegram-бота**, чтобы пользоваться из любой сети
(LAN не нужен). Если хостер ещё не запустил — это один самодостаточный
файл (`deltav/tgbot.py`), long-polling, без входящих портов:

```bash
# это запускает хостер (токен даёт BotFather):
TELEGRAM_BOT_TOKEN=… DELTAV_GATEWAY=http://<gw>:9000 \
  DELTAV_ALLOW=<ваш-telegram-id> python -m deltav.cli tgbot
```

Дальше в вашем боте: просто пишите (агент отвечает, ищет в интернете
когда нужно, помнит вас); **пришлите фото** 🖼 — спросить по картинке;
команды `/agent <задача>`, `/models`, `/net`, `/plan`, `/fast`, `/reset`.

## Как получить API-ключ (оплата)

Ключ `dvk_…` — это кастодиальный он-чейн кошелёк: пополните его адрес
один раз, и каждый запрос оплачивается с него. Переплатить нельзя —
ценовой лимит подписывается на каждый запрос.

**Нужен ли он вообще?** В приватной / LAN-сети шлюз может платить со
своего кошелька (ключ не нужен). Ключ требуется, когда шлюз запущен с
`--require-keys`, или когда хотите свою отдельную оплату.

**Как получить** — любым способом:

```bash
# а) через веб-приложение: откройте http://<gw>:9000/chat  (для пробы ключ не обязателен)
# б) создать через API (ключ показывается ОДИН раз — сохраните):
curl -X POST http://<gw>:9000/v1/keys
#    -> {"api_key":"dvk_…","address":"dv1…","note":"пополните этот адрес"}
```

Затем **пополните адрес ключа** токенами DVT — попросите хостера сети
отправить немного на полученный `address` (он выполнит
`deltav send --to dv1… --amount 100`), либо переведите со своего кошелька.
Проверить:

```bash
deltav keys me --key dvk_…         # баланс + расход
```

## Быстрая проверка

```bash
curl http://<gw>:9000/v1/chat/completions \
  -H "Authorization: Bearer dvk_…" -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"привет"}]}'
```

Поле `deltav` в ответе показывает, какая нода вас обслужила, и чек в
чейне — его видно в эксплорере (`http://<нода>:9100/explorer`).

## Заметки

- Веб-приложение и прямой API — **только в той же сети** (LAN). Для
  доступа откуда угодно хостер ставит туннель (Cloudflare/Caddy TLS)
  перед шлюзом — тогда используйте его публичный адрес. Telegram работает
  из любой точки.
- **Reasoning-модели** по умолчанию отвечают напрямую; если ответы
  выглядят обрезанными — поднимите `max_tokens`.
- **Зрение** требует, чтобы обслуживалась vision-модель (помечена 👁 в
  выборе модели / `deltav models`).
