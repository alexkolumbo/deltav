# Запуск ноды Delta V (хостинг)

[English](HOSTING.md) · [Русский](HOSTING.ru.md)

Нода — это компьютер, который отвечает на запросы к ИИ для сети и
зарабатывает токены DVT. Подойдёт любая машина — игровой ПК, ноутбук,
сервер — на **AMD, NVIDIA, Intel, Apple или CPU**. Это руководство ведёт
от нуля до живой зарабатывающей ноды.

## Что нужно

- Компьютер с **Python 3.11+** (Windows, Linux или macOS).
- Идеально — **видеокарта с ≥6 ГБ VRAM** (любой вендор); CPU тоже
  работает, но медленно.
- Адрес одной **живой ноды** для подключения (seed), либо вы запускаете
  первую.
- ~5–15 ГБ на диске под модель.

## Просто — одна команда

```bash
# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/alexkolumbo/deltav/main/install.sh | sh
# Windows (PowerShell)
irm https://raw.githubusercontent.com/alexkolumbo/deltav/main/install.ps1 | iex
```

Запустится **мастер установки** (двуязычный). Он, объясняя каждый шаг:

1. **Определит железо** (видеокарта или CPU, сколько памяти).
2. **Подберёт модель** под вашу VRAM (ранжирует из БД моделей; можно
   принять, выбрать другую или вставить свой HuggingFace-репо — с
   анализом влезаемости).
3. **Скачает движок** — готовый бинарник `llama.cpp` под вашу ОС+GPU
   (Vulkan для AMD/NVIDIA/Intel, Metal для Apple, CPU-фолбэк). Ничего
   не компилируется, CUDA/ROCm не нужны.
4. **Скачает модель.**
5. **Заведёт кошелёк** — адрес для заработка. Сохраните этот файл.
6. **Подключит к сети** — заберёт генезис у seed-ноды.
7. **Установит цену** — по себестоимости (электричество + 50%) или свою.
8. **Запустит** — поднимет движок + ноду и покажет адрес панели.

Ещё запишет скрипт `start-node`, чтобы в следующий раз запускать одной
командой.

## Если Python уже есть

```bash
pip install "deltav-network[gpu,hub] @ git+https://github.com/alexkolumbo/deltav"
deltav setup --seed http://<любая-живая-нода>:9100
```

## Ручной контроль (`deltav join` / `deltav node`)

Полный контроль над моделью, бэкендом и пирами:

```bash
# 1) локальный llama.cpp-сервер с вашей моделью (Vulkan-сборка = любой GPU)
llama-server -m <model.gguf> --host 127.0.0.1 --port 8085 -ngl 99 -c 8192
#    модель со зрением? добавьте:  --mmproj <mmproj.gguf>
#    reasoning-модель?  размышления выключены по умолчанию; DELTAV_THINK=1 включит

# 2) нода, указывающая на этот сервер
deltav node --genesis genesis.json --wallet node.wallet.json \
    --host 0.0.0.0 --port 9100 --endpoint http://<ваш-ip>:9100 \
    --peer http://<seed>:9100 \
    --backend llamaserver --model "<org/repo::file.gguf>" \
    --data-dir ./data --price 9
```

`deltav join --seed http://<node>:9100` сделает шаги железо→модель→
скачивание→запуск автоматически, если не хотите выбирать всё руками.

## Запуск самой первой ноды (своя сеть)

```bash
deltav wallet new --file node.wallet.json
deltav genesis --alloc dv1...=100000 --stake dv1...=10000 \
    --dev-fund dv1... -o genesis.json
deltav join --genesis genesis.json --wallet node.wallet.json --port 9100
# затем шлюз, чтобы клиенты могли подключаться (OpenAI/Ollama/Anthropic):
deltav gateway --genesis genesis.json --node http://127.0.0.1:9100 --port 9000
```

## Сделать доступной (внешний доступ)

Другие ноды достучатся до вас по публичному адресу. `deltav join` определит
его сам — **без проброса портов, без ручного IP, без настройки TLS**:

- **`--connect auto`** (по умолчанию для `join`) — нода узнаёт свой публичный
  адрес у пира, сама проверяет, доступна ли она напрямую, и если да —
  анонсирует его. За NAT/CGNAT вместо этого открывает **исходящий** туннель к
  публичной ноде-релею и получает публичный URL `https://<relay>/via/<node-id>` —
  доступна с **нулём входящих портов**.
- **`--connect direct`** — форсировать определённый публичный адрес (у вас
  публичный IP или проброшен порт). Откройте порты:

  ```powershell
  # Windows (PowerShell от админа) — нода 9100, шлюз 9000
  New-NetFirewallRule -DisplayName "DeltaV" -Direction Inbound `
    -Protocol TCP -LocalPort 9000,9100 -Action Allow
  ```
  ```bash
  # Linux (ufw)
  sudo ufw allow 9000,9100/tcp
  ```

- **`--connect relay`** / **`--relay-via <url>`** — всегда через релей
  (удобно на CGNAT или ноутбуке, который меняет сети).
- **`--connect local`** — только LAN / разработка; анонсировать `host:port`
  как есть.

По-прежнему можно задать `--endpoint http://<адрес>:9100` вручную — он всегда
принимается как есть.

### Поднять релей (помочь NAT-нодам подключиться через вас)

Любая нода с публичным адресом может стать релеем, чтобы ноды за NAT
достучались до сети через неё — именно это держит внешний доступ
**децентрализованным**, без стороннего туннель-сервиса:

```bash
deltav node ... --relay --relay-url https://relay.example.com
```

Релей пересылает только **подписанный** трафик — он не может прочитать или
подделать сообщения чейна (каждая транзакция, блок и чек подписаны) и
туннелирует только для ноды, криптографически доказавшей владение своей
личностью. Поставьте TLS перед публичным релеем (ваш Caddy / обратный
прокси), чтобы релейные ноды отдавали по `https://`.

**Безопасность из коробки:** госсип ограничен по частоте на IP, размер тела
запроса ограничен; пир-URL, узнанный из госсипа, принимается только после
доказательства, что он в вашем чейне; оплата всегда требует подписи
реквестера над ценовым лимитом. Следите за нодой на **`http://<адрес>:9100/explorer`**.

## Выбор модели

```bash
deltav plan                       # лучшие модели под ваше железо + команды запуска
deltav registry list --vram 8176  # полная ранжированная БД (каталог + HuggingFace)
deltav registry sync              # подтянуть свежие модели с HuggingFace
deltav registry add <org/repo>    # добавить конкретную HF-модель в БД
```

На 8 ГБ: Qwen2.5-7B (32k контекста) — отличный практичный дефолт;
Llama-3.2-3B тянет 128k контекста; reasoning/vision-модели (DeepSeek-R1,
Qwythos-9B) работают, но много думают и влезают впритык.

## Цена и заработок

```bash
deltav price --watts 130 --tps 30 --kwh-usd 0.10   # ваша рекомендованная цена
```

Цена заякорена в себестоимости: электричество + 50% сервиса, по
среднемировой цене электричества по умолчанию. Дешевле энергия или
быстрее нода → ниже цена → больше трафика. Платят в DVT за токен через
чеки в чейне; 10% каждого платежа идёт в пул сети, распределяемый каждую
эпоху нодам за работу и в фонд разработчиков.

## Держать в работе

- Чейн персистится в `--data-dir`; рестарт восстанавливает его с диска.
- На Linux запускайте `start-node` под `systemd` или `tmux`.
- На Windows `start-node.bat` из мастера заново поднимает движок + ноду.
- Следите за нодой на **`http://<ваш-ip>:9100/explorer`** — высота,
  загрузка, репутация, чеки, заработок.

## Решение проблем

| Симптом | Решение |
|---|---|
| Мастер: «нет готового бинарника» | Поставьте `llama.cpp` вручную и используйте `deltav node` |
| Нода не зарабатывает | Проверьте анонс: `deltav network`; убедитесь, что шлюз на неё маршрутизирует |
| Модель отвечает пусто | Reasoning-модель — размышления выключены по умолчанию; поднимите `--max-tokens` у клиента или `DELTAV_THINK=1` на сервере |
| Зрение не работает | Запустите `llama-server` с `--mmproj <projector.gguf>` |
| Out of memory при загрузке | Возьмите модель меньше (`deltav plan`) или уменьшите `-c` (контекст) |
| Рестарт потерял чейн | Он ресинкается с `--peer`; убедитесь, что пир задан |
